# Решение проблемы с типами TypeScript

Если вы видите ошибки TypeScript о том, что свойство `contact` не существует, это проблема кэша TypeScript сервера в IDE.

## Решение:

1. **Перезапустите TypeScript сервер в VS Code:**
   - Нажмите `Ctrl+Shift+P` (или `Cmd+Shift+P` на Mac)
   - Введите "TypeScript: Restart TS Server"
   - Нажмите Enter

2. **Или перезапустите VS Code полностью**

3. **Проверьте, что все файлы переводов имеют свойство `contact`:**
   - `src/locales/en.ts` ✅
   - `src/locales/ru.ts` ✅
   - `src/locales/kz.ts` ✅

Все файлы переводов содержат свойство `contact` со всеми необходимыми полями.

## Проверка:

Запустите в терминале:
```bash
npm run build
```

Если сборка проходит успешно, значит код правильный, и проблема только в кэше IDE.
